En un kernel en CUDA,
3
- Cada hilo de un grid ejecuta un código distinto.
- Cada bloque de un grid ejecuta un código distinto.
- Todos los hilos agrupados en un grid ejecutan el mismo código.

¿Qué es un kernel en la GPU?
2
- Es la función que se ejecutará en los distintos hilos de la GPU de manera secuencial
- Es la función que se ejecutará en los distintos hilos de la GPU de manera simultánea
- Es la función que se ejecutará con un modelo MISD en los distintos hilos de la GPU

¿Qué especificador en la declaración nos permite definir la función vecadd para que se ejecute en la GPU?
1
- __global__ void vecadd()
- __host__ void vecadd()
- __kernel__ void vecadd()

¿Qué hace la siguiente función? ```__global__ void vecadd(float* C, const float* A, const float* B)\n{ \n\t# ================================================================ \n\t# Calcula el índice para acceder a cada elemento \n\t# Calcula la suma de las posiciones correspondientes y almacena el resultado \n\t# ================================================================\n\n\tconst int i = (threadIdx.x + blockIdx.x * blockDim.x) * COMPUTE_N_ELEMENTS_PER_THREAD; \n\tif ( i < VECTOR_ELEMENTS ) \n\t{ \n\t\tfor(int j=0; j < COMPUTE_N_ELEMENTS_PER_THREAD; j++) \n\t\t{ \n\t\t\tC[i+j] = A[i+j] + B[i+j]; \n\t\t} \n\t}\n}```
3
- Es una función global que realiza la suma en el host de las VECTOR_ELEMENTS componentes del vector.
- Es el Kernel que se ejecuta en paralelo por cada hilo en la GPU para calcular la suma de las VECTOR_ELEMENTS componentes del vector. Cada hilo realiza la suma del número de componentes del vector indicado por COMPUTE_N_ELEMENTS_PER_THREAD.
- Es el Kernel que se ejecuta en paralelo por cada hilo en la GPU para calcular la suma de las VECTOR_ELEMENTS componentes del vector. Cada hilo solo puede realizar una única suma.

¿Qué hace el siguiente fragmento de código?```# ===================================================\n# Calcula las dimensiones del grid e invoca el kernel\n# ===================================================\ndim3 block(512);\ndim3 grid( (VECTOR_ELEMENTS + (block.x*COMPUTE_N_ELEMENTS_PER_THREAD-1)) / (block.x*COMPUTE_N_ELEMENTS_PER_THREAD) );\n\n#ifdef DEBUG\n\tprintf(" # of threads in a block %d/n", block.x);\n\tprintf(" # of blocks in a grid %d/n", grid.x);\n\tprintf(" Executing the kernel.../n");\n#endif\n\ngpu_start_time = getTime();\n\nvecAdd<<<grid,block>>>(C_d,A_d,B_d);\n\ncudaThreadSynchronize();\ngpu_end_time = getTime();\nERROR_CHECK\n```
1
- Define la estructura y dimensión de los bloques y del grid e invoca al kernel. La estructura definida para los bloques es de una dimensión.
- Define la estructura y dimensión de los bloques y del grid pero no se invoca ningún kernel. La estructura definida para los bloques es de una dimensión.

Sobre la complejidad y dimensión de la cache, la unidad de control y las ALUs de la CPU frente a la GPU ¿Cuál de las siguientes afirmaciones es cierta?
1
- Las memorias caches de la GPU son individualmente de mayor tamaño que la cache de la CPU
- Las ALUs de la CPU son más complejas y mucho menos numerosas que las ALUs de la GPU
- La unidad de control de la CPU es mucho más simple que las unidades de control de la GPU

EN CUANTO A LAS DIFERENCIAS ENTRE LA CPU Y LA GPU:
2
- Una CPU multinúcleo necesita unos pocos hilos más que la GPU para ser igual de eficiente
- Los hilos de la GPU son más ligeros que los de la CPU, necesitando más hilos que esta última para ser eficiente.
- Las latencias de memoria en la CPU son mayores que en la GPU.
